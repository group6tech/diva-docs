Adding a DIVA User Account

When adding a DIVA user account, you can grant the user access to one or both of the following:

  • DIVA Client
  • DIVA Fusion

Before You Begin

Before adding a user account, ensure:

  • DIVA Fusion is running and you are signed in
  • you have the user's first name and last name
  • the user has chosen or been assigned a user name for the user account, and
  • (optional) you have the user's email address.

To add a DIVA user account:

  1. From DIVA Fusion's Users menu, click Users.
    • Result: A list of users is displayed.
  2. Click New User.
    • Result: The User Information and Member Information fields are displayed.
  3. Complete all relevant user Information and Member Information fields and options.
    • Note: The fields marked with a red asterisk are mandatory.
    • Note: With the Create Member option selected (ticked), the user will be able to sign into DIVA Fusion.
    • Note: With the DIVA User option selected (ticked), the user will be able to sign into DIVA Client and access the drives. You can restrict access to drives later on.
  4. Click Save User
    • Result: Assuming the user account does conflict with another user account, the user account is created.
  5. Select (tick) the 'Enable User Account' option.
    • Note: If selected, the user account will be enabled and will be ready to use. If deselected, the user account will be disabled and the user will not be able to use it to sign into a DIVA product.
  6. Click Save User.
    • Result: The changes you made are saved, and the user account is enabled.

What Next?

Grant the user access to one or more drives. To do this see 'Modifying a User's Access to One or More Drives'

Modifying a User's Access to One or More Drives

You can:

  • deny the user access to one or more drives
  • Note: When new user accounts are set up, access to drives is automatically denied.
  • grant read-only access to one or more drives
  • Note: The user can view, and copy files to another drive, but cannot save this files to read-only drive.
  • grant read/write access to one or more drives
  • Note: The user has full access to the drive and can save this files to it.

Granting a User Permission to Save Files (Write) to a Drive

NOTICE! Moving Users' Media
If you choose to restrict users' access to an Avid Media drive, you will be given the option to move the users' media to another user account. Proceed carefully. Once the media is moved, this cannot be undone.

Before You Begin

Before modifying a user's access to one or more drives, ensure:

  • DIVA Fusion is running and you are signed in
  • an account for the user has been added, and
  • Note: To do this, see 'Adding a DIVA User Account'
  • the drives that you are going to grant the user access to have been set up.
  • Note: If you need to set up the drives, for assistance, see Drives.

To modify a user's access to one or more drives:

  1. From DIVA Fusion's Users menu, click Users.
    • Result: A list of users is displayed.
  2. From the list, click the relevant user account.
    • Result: The user account's properties are displayed.
  3. Click Drives.
    • Result: A list of drives is displayed.
  4. For the drives in question, select the relevant options.
    • Note: Options include Denied, Read and Write.
    • Note: You cannot mount a drive for the user from this page. If there is a tick, the user has mounted the drive and you cannot make changes. If there is a cross, the drive is not mounted for that user.
  5. Click Save Permissions.
    • Result: The changes you made are saved.

What's Next?

Remember to notify the user of the changes.

Enabling a User Account

If the account was disabled for any reason, e.g., setting up an account in advance for an upcoming project, you can enable it at any time.

Before You Begin

Before enabling a user account, ensure DIVA Fusion is running and you are signed in.

To enable a user account:

  1. From DIVA Fusion's Users menu, click Users.
    • Result: A list of users is displayed.
  2. From the list, click the relevant user account.
    • Result: The user account's properties are displayed.
  3. Select (tick) the 'Enable User Account' option.
  4. Click Save User.
    • Result: The changes you made are saved, and the user account is enabled.

Performing a Health Check on a User Account

This process checks the selected users' details, e.g., password, for consistency across all DIVA nodes. Perform this health check whenever a user is having problems using DIVA Client.

Before You Begin

Before performing a health check on a user account, ensure:

  • ensure DIVA Fusion is running and you are signed in, and
  • the users whose accounts you want to perform a health check on, are signed out.
  • Note: Out of courtesy, please notify the user before disconnecting the user's account.

To perform a health check on a user account:

  1. From DIVA Fusion's Users menu, click Users.
    • Result: A list of users is displayed.
  2. Select the relevant user account.
    • Note: This is the user account that you want to perform a health check on.
    • Note: You can select more than one user account.
  3. If the user is signed in (online), disconnect the user account.
    • Note: To disconnect the selected user's account, click Disconnect.
    • Note: The user is automatically warned through DIVA Client that server is shutting down and that they will be disconnected.
  4. Click Health Check.
    • Result: You will be informed that the health check has started.

What's Next?

To see if the health check is complete, on the DIVA Fusion menu, click System, then click Jobs. The job will be in the list. It will either be queued, running or completed

Disabling a User Account

Before You Begin

Before disabling a user account, ensure DIVA Fusion is running and you are signed in.

To disable a user account:

  1. From DIVA Fusion's Users menu, click Users.
    • Result: A list of users is displayed.
  2. From the list, click the relevant user account.
    • Result: The user account's properties are displayed.
  3. Deselect (remove tick) the 'Enable User Account' option.
  4. Click Save User.
    • Result: The changes you made are saved, and the user account is disabled.

Forcing a User to Sign Out of DIVA Client

If, for example, you need to reboot a node, or the user is signed out from an unknown location, you can force the user's user account to sign out of DIVA client.

NOTICE! Forcing a User Account to Sign Out of DIVA Client
Only force a user to sign out of DIVA Client as a last resort. When forcing the user to sign out, the drives will be unmounted and data loss may occur.

Before you begin

Before forcing a user to sign out of DIVA client where practicable, let the user know that they may need to save their work.

To force a user to sign out of DIVA Client:

  1. From DIVA Fusion's Users menu, click Users.
    • Result: A list of users is displayed.
  2. From the list of user accounts, click the relevant checkboxes.
    • Result: The users' accounts are selected.
  3. Click Disconnect.
    • Result: The users will be notified that their user accounts are to be disconnected within ten seconds. After ten seconds the selected user accounts are disconnected.

Deleting a User Account

Only delete a user account if you are certain it will no longer be needed. If it is possible that the user will require the account again, you can disable the user account for now.

NOTICE! Deleting a User Account
Before deleting the user account, ensure all of the user's files that you want to keep have been moved to another user account. Deleting a user account automatically deletes the user's private projects and Avid media files. Their shared project files will be moved to Unprotected.

Before You Begin

Before deleting a user account, ensure DIVA Fusion is running and you are signed in.

To delete a user account:

  1. From DIVA Fusion's Users menu, click Users.
    • Result: A list of users is displayed.
  2. From the list, click the relevant user account.
    • Result: The user account's properties are displayed.
  3. Click Delete User.
    • Result: You will be asked to confirm that you want to delete the user account.
  4. In the Confirmation field, type DELETE.
    • Note: This confirmation message is only displayed if the user has read or write access to any drive.
    • Note: If all user has been denied access to all drives, the Confirmation field is not displayed.
  5. Click Delete User.
    • Result: The user account is deleted.

Resetting a User's Password

Before You Begin

Before resetting a user's password, ensure DIVA Fusion is running and you are signed in.

To reset a user's password:

  1. From DIVA Fusion's Users menu, click Users.
    • Result: A list of user accounts is displayed.
  2. From the list, click the relevant user account.
    • Result: The user account's properties are displayed.
  3. Click Membership.
  4. Click Change Password
    • Result: Two password fields are displayed.
  5. In the New Password and Confirm Password fields, type the new password.
  6. Click Change Password.
    • Result: The password is changed.

What's Next?

Remember to notify the user of the new password.

System User Groups

System user groups control access to the different areas of DIVA Fusion.

Server Administrator

A Server Administrator can:

  • create nodes
  • connect nodes, and
  • manage the job queue.

DIVA Administrator

A DIVA administrator can manage:

  • assign volumes to a license pool
  • remove a license from a license pool
  • add, edit and delete favourites
  • manage projects, and
  • modify users' access to drives (permissions)

User Administrator

A User Administrator can:.

  • add, enable, disable, modify and delete user accounts, and
  • modify users' access to drives (permissions)

Group Administrator

A Group Administrator can:

  • create, edit and delete user groups, and
  • assign user accounts to user groups.

Administrator

An Administrator can do all of the above. That is, an administrator can do everything a:

  • Server Administrators can do
  • DIVA Administrator can do
  • User Administrator can do, and
  • Group Administrator can do.

Setting Up a User Group

If you want to create a favourite for a specific set of DIVA Client users to use, e.g., a project team, first you need to set up a user group and assign individual user accounts to it.

Before You Begin

Before setting up a user group, ensure DIVA Fusion is running and you are signed in.

To set up a user group:

  1. From DIVA Fusion's Users menu, click Groups.
    • Result: A list of user groups is displayed.
  2. Click New Group.
    • Result: The group's properties are displayed.
  3. In the Group Name field, type the name of the group.
    • Note: This is the name that you want to give the group.
  4. (Optional) In the Description field, type a description of the group.
    • Note: Provide enough information to distinguish this group from any others.
  5. Click Create New User Group
    • Result: The group is created and a Members tab is added next to the Properties tab.
  6. Click Members.
    • Result: A drop down menu of user accounts is displayed.
  7. From the drop down menu, select the user account that you want to add to the group.
  8. Click '+Add Member'
    • Result: A table lists the group's new user account.
  9. Do you want to add another user account to the group?
    • If Yes, go to Step 7.
    • If No, go to What's Next? (below).

What's Next?

Click All Groups to see your new group listed along with the other groups.

Removing a User Account from a User Group

Before You Begin

Before removing a user account from a user group, ensure DIVA Fusion is running and you are signed in.

To remove a user account from a user group:

  1. From DIVA Fusion's Users menu, click Users.
    • Result: A list of user accounts is displayed.
  2. Click the relevant user account.
    • Result: The user account's properties are displayed.
  3. Click Groups.
    • Result: The groups to which the user belongs are listed at the bottom of the page. A small cross is displayed against the groups from which the user account can be removed.
  4. Click the relevant user's cross.
    • Result: The user is removed from the group.
