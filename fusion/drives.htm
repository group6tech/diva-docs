About Drives

Drive Types

If mounted on user's computer and the user has been granted full access to a drive, the drive is available to save files and work on. If the user has only been granted read-only access to the drive, the user can only view files and copy files from the read-only drive to another drive.

Note: These drives are not physical storage devices; they are virtual drives that reside on the volume (storage device).

Drive Types

The picture to the right shows a volume (hard disk). In this example, the volume has been assigned to an Avid Pool and three drives have been set up; one of each type.

There are three types of drive:

  • Basic Drive
  • Avid Media Drive.
  • Avid Projects Drive

Basic Drives

If a user has full access to a Basic drive, the user can add files of any type.

Note: With Avid, you can't create Avid MXF files on a Basic drive.

Media Drives

Media drives are optimised exclusively for Avid MFX files. To share Avid MXF media files, the media has to be created on a Media drive.

Users can only save and work with Avid MXF files on this drive.

Projects Drives

Projects drives are optimised exclusively for Avid Media Composer projects and bins. To share an Avid project, it has to be created on a Projects drive.

Note: With Avid, you can't create Avid MXF files on a projects drive.

Adding a Drive to a Volume

The following diagram identifies the drives that can be added to a particular volume.

Drives & Volumes

If the volume is assigned to an Avid Pool, drives of any type (Basic, Media & Project) can be added to it. If the volume is assigned to a Basic Pool, only Basic Drives can be added to the volume.

Note: These drives are not physical storage devices; they are virtual drives that reside on the volume (storage device). When mounted by a user on a Windows PC, the drive is assigned a drive number.

Note: On Basic drives and projects drives, using Avid, you cannot create Avid MXF files.

Note: You can only add a Project drive to a media or projects volume if it is permitted for that volume. To permit or prevent project drives from being added to a volume, see 'DIVA Fusion > Volumes > Permitting or Preventing the Addition of Avid Project Drives on Avid DIVA Volumes'.

Before You Begin

Before adding a drive to a volume, ensure DIVA Fusion is running and you are signed in.

To add a drive to a volume:

  1. From DIVA Fusion's Drives menu, click Drives.
    • Result: The DIVA Drives page is displayed.
  2. Click New Drive.
    • Result: The Properties tab is displayed.
  3. From the Volume drop down menu, select the relevant volume.
  4. In the Drive Name field, type the name that you want to call this drive.
    • Note: You can use uppercase and lowercase letters, numerals, dashes and underscores. Do not use slashes or any other special characters.
  5. Select the relevant drive type.
    • Note: Ensure the select drive type correlates with the selected volume's license.
    • Note: You can't add a projects drive to a Basic volume. You can only add a new projects drive to a Media or Projects volume if this has been permitted for the volume.
  6. Ensure the Enabled checkbox is selected (ticked).
    • Note: If you deselect this option, users will not be able to access the drive.
  7. Click Save Drive.
    • Result: The drive is added to the volume.

What's Next?

  • Grant a number of users access to this drive.
  • Grant one user access to multiple drives, including this one.

Modifying Access to a Drive for One or More Users

If you have, for example, added a new drive, you can grant a range of users access to the drive from the same page. This means you don't have to modify user accounts one by one. At any time, you can select a drive, and change everyones access in one go.

You can:

  • deny one or more users access to the drive
  • Note: When new drives are set up, access to drives is automatically denied.
  • grant one or more users read-only access the drive
  • Note: With read-only access, users can view, and copy media to another drive, but cannot save any files to a read-only drive.
  • grant one or more users read/write access to the drive
  • Note: With full access users can save media to the drive.

If you reduce a user's access from Full access, you will be asked if you want to move the user's media to another user account. You can choose to:

  • move the user's media to another user account
  • leave the user's media where they are, or
  • Note: If you reduce the user's access to read-only access and leave the files on the drive, the user will be able to view the files and copy them to another drive. If you deny the user access to the drive, the user will lose all access to the files. You can reinstate access at any time.
  • cancel your changes
  • Note: If you cancel your changes, the changes you made onscreen will not be saved. The page will still show your changes, but they are not saved. You can continue to make changes before saving them.

Denying a User Access to a Drive

NOTICE! Moving Users' Media
If you choose to restrict users' access on an Avid Media drive, you will be given the option to move the users' media to another user account. Proceed carefully. Once the media is moved, this cannot be undone.

End of Project

At the end of a project, you may want to initially prevent users from modifying files by denying them access to the drive, or by only providing them with Read access. Then after a month or so after you have archived off the files, you can delete the drive.

Before You Begin

Before modifying access to a drive for one or more users, ensure:

  • DIVA Fusion is running and you are signed in
  • the user accounts have been added, and
  • Note: To do this, "User Accounts > Adding a DIVA User Account"
  • the drive exists.
  • Note: If you need to add a drive, see Adding a Drive.

To modify access to a drive for one or more users:

Note: If you have just created a drive and its Properties pane is still displayed, go straight to Step 3.

  1. From DIVA Fusion's Drives menu, click Drives.
    • Result: A list of drives is displayed.
  2. Click the relevant drive.
    • Note: You will be modifying users' access to the selected drive.
    • Result: The drive's properties are displayed.
  3. Click Permissions.
    • Result: A list of user accounts is displayed.
  4. For each user account (where relevant), select the relevant access options.
    • Note: Options include Denied, Read and Write.
    • Note: To apply the same setting to all users in the list, click the column title (click Denied, click Read or click Write).
    • Note: If you intend to downgrade more than one user's access, you will be requested to move all the users' files to one user account. If you intend to move the users' files to more than one account, repeat Steps 4 to 6, for each account that you want to move the files to.
    • Note: If you lose track of what you have changed, you can click Undo Changes to reinstate the previously saved permissions.
    • Note: If you inadvertently select the user's name, the user's account is displayed. Immediately click the browser's back button to redisplay the list of user accounts.
  5. Click Save Permissions.
    • If this is a Basic Drive or a Avid Projects drive, go to 'What's Next' below.
    • If this is an Avid Media drive, and you did not reduce anyone's access, your changes are saved. Go to 'What's Next' below.
    • If this is an Avid Media drive and you reduced one or more users' access to the drive, you will be presented with options for moving the relevant users' files. If this is the case, go to Step 6.
  6. Make the relevant choices.
    • Note: To move all listed users' files to one user account, from the Move To drop down menu, select the user account that you want to move the files to, then click Move Media.
    • Note: To leave the files where they are, click 'Leave in Place'.
    • Note: If you don't want to save your changes, click Cancel.



    • Result: On clicking 'Move Media', the media files are moved. On clicking 'Leave in Place', your changes are saved and the media files will remain with the user accounts.
    • Note: On clicking Cancel, your changes will not be saved and you can continue to select other options for each user. In this case, go to Step 4.

What's Next?

Remember to notify the users of the changes that you have made.

Adding a Drive Set

With DIVA Fusion, you can add a set of new drives consisting of all three drive types, and you can do this in one go. This means you don't have to add the drives one-by-one. All new drives in the set are automatically suffixed, making them easy to identify. If you want to, you can modify the suffix to suit your drive naming conventions. As part of the same process, you can assign drive access to your users.

Before You Begin

Before adding a drive set, ensure DIVA Fusion is running and you are signed in.

To add a drive set:

  1. From DIVA Fusion's Drives menu, click Drives.
    • Result: The DIVA Drives page is displayed.
  2. Click New Drive Set.
    • Result: The New Drive Set page is displayed.


  3. In the Base Name field, type the name that you want to see on all three drives.
    • Note: You can use uppercase and lowercase letters, numerals, dashes and underscores. Do not use slashes or any other special characters.
  4. (Optional) Deselect (untick) the Create Favourite checkbox.
    • Note: Only deselect this if you don't want a favourite set up for users. If you want a favourite, ensure the checkbox is selected (ticked)
  5. (Optional) Deselect (untick) the drives that you don't want.
    • Note: Only do this if you don't want all three drives in the set.
  6. (Optional) Select other volumes for your drives.
    • Note: Your drives don't all have to reside on the same volume. If you think the volume may have insufficient space for all the media files that will be created, select another volume.
  7. (Optional) Overtype each suffix with the suffix you want to see on the drive name.
    • Note: Each drive name will consist of the base name and a suffix.
  8. For each user account (where relevant), select the relevant access options.
    • Note: Options include Denied, Read and Write.
    • Note: To apply the same setting to all users in the list, click the column title (click Denied, click Read or click Write).
    • Result: Your updated page should look something like this:


  9. Click Create Drive Set.
    • Result: The drive set is added to the volume/s you selected and listed on the DIVA Drives page.



What's Next?

To see if everything has been set up, you can:

  • check the job queue, and
  • Note: To do this, from DIVA Fusion's Server menu, click Jobs.
  • Note: The job queue shows you the status of all the jobs that were generated on clicking Create Drive Set.
  • check to see if the drives can be mounted.
  • To do this, open DIVA Client and sign in as one of the users to mount a drive. If DIVA Client is not already open, on the desktop, double-click the DIVA Client shortcut.

Setting a Drive Capacity Quota

Sometimes you may want to restrict the capacity of a drive. The limit that you can set will depend on the size of the volume.

Quota Field & Slider

Before You Begin

Before setting a drive capacity quota, ensure DIVA Fusion is running and you are signed in.

To set a drive capacity quota:

  1. From DIVA Fusion's Drives menu, click Drives.
    • Result: The DIVA Drives page is displayed.
  2. Locate and click the relevant drive.
    • Note: If the list of drives is long, use the search tools on the left to narrow down your search.
    • Result: The drive's properties are displayed. The Quota field displays the maximum amount of disk space this drive can use.
    • Note: You may notice that the quota is at its maximum, the free space is less than the quota. This is because some space is reserved for system use.
  3. In the Quota field, overtype the existing value, and from the drop down menu, select the relevant units.
    • Note: Available units are GB (Gigabytes) and TB (Terabytes).
    • Note: Alternatively, you can use the slider to adjust the quota.
    • Note: You cannot specify a value that exceeds the volume's capacity.
  4. Click Save Drive.
    • Result: Your changes to the quota are saved. The amount of free space is also updated to reflect the change in quota.

What's Next?

To see if everything has been set up, you can:

  • check the job queue, and
  • Note: To do this, from DIVA Fusion's Server menu, click Jobs.
  • Note: The job queue shows you the status of all the jobs that were generated on clicking Save Drive.
  • check the space available to the user has been updated.
  • Note: To do this, open DIVA Client and sign in as one of the users to mount a drive. If DIVA Client is not already open, on the desktop, double-click the DIVA Client shortcut. Also, browse (Window's Explorer or Finder) to the mounted drive and check the available space.

Deleting a Drive

When you no longer have a need for the drive, you can delete it.

NOTICE! Deleting Drives
Back up or archive any files the you want to keep before deleting a drive. Once deleted, the drive's data cannot be recovered and will be lost.

Before you begin

Before deleting a drive, ensure:

  • DIVA Fusion is running and you are signed in
  • you have backed up or archived any files that you want to keep for future use, and
  • the drive is not currently mounted by users.

To delete a drive:

  1. From DIVA Fusion's Drives menu, click Drives.
    • Result: The DIVA Drives page is displayed.
  2. Locate and click the relevant drive.
    • Note: If the list of drives is long, use the search tools on the left to narrow down your search.
    • Result: The drive's properties are displayed.
  3. At the bottom of the page, click Delete Drive.
    • Result: You will be asked to confirm that you want to delete the drive.
  4. In the Confirmation field, type DELETE.
    • Note: This confirmation message is only displayed if any users have read or write access to the drive.
    • Note: If all users have been denied access, the Confirmation field is not displayed.
  5. Click Delete Drive.
    • Result: The drive and all its files are deleted.

What's Next?

To see if the drive has been deleted, you can check the job queue.

Note: To do this, from DIVA Fusion's Server menu, click Jobs.

Note: The job queue shows you the status of all the jobs that were generated on clicking Delete.

Performing a Health Check

A health check will clean up permissions and will re-enable the drive. If anyone has the drive mounted, it cannot be checked. A health check can take anything from seconds to hours to complete depending the amount of data on the drive. Please allow up to an hour on average.

The health check takes about 1 minute per TB, or 1 minute per 10,000 files, whichever is higher. For example:

  • for a 5 terabyte drive with 2,000 files, allow 5 minutes, and
  • for a 3 terabyte drive with 80,000 files, allow 8 minutes.

NOTICE! Health Checks
Only perform a health check on a drive when you are certain your users have no need to use the drive. When a health check is in progress, the drive cannot be mounted and other jobs on the node, e.g., adding drives to the same node, cannot be initiated.

Before You Begin

Before performing a health check, ensure DIVA Fusion is running and you are signed in.

To perform a health check:

  1. From DIVA Fusion's Drives menu, click Drives.
    • Result: The DIVA Drives page is displayed.
  2. Locate and click the relevant drive.
    • Note: If the list of drives is long, use the search tools on the left to narrow down your search.
    • Result: The drive's properties are displayed.
  3. At the bottom of the page, from the Tools drop down menu, click Health Check.
    • Result: The health check begins.
    • Note: This may take an hour on average. The drive will be in maintenance mode and cannot be used until the health check is complete.

What's Next?

When the health check is complete. The drive will be ready for mounting by users.

What's Next?

To see if the drive has been deleted, you can

  • Check the job queue
  • Note: To do this, from DIVA Fusion's Server menu, click Jobs.
  • Note: The job queue shows you the status of all the jobs that were generated on clicking Delete Drive.
  • When the job is complete, the drive will return to online status and can be mounted by users.
